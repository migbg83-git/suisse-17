# Archwise Publishing Checklist

## Fase editorial
- [ ] brief.md completado
- [ ] notes.md completado
- [ ] outline.md completado
- [ ] article.md con frontmatter YAML válido
- [ ] review.md revisado y checklist de calidad
- [ ] assets.md con prompts, imágenes y Open Graph

## Fase publicación
- [ ] article.json generado y validado
- [ ] Ejecutar `npm run build:content`
- [ ] Ejecutar `npm run build:ssg`
- [ ] Verificar `/articulos` y la página individual
- [ ] Verificar `sitemap.xml`
- [ ] Verificar Open Graph en la URL
- [ ] Verificar Analytics registra visitas
- [ ] Verificar Search Console indexa el artículo

## Validaciones finales
- [ ] URL accesible y correcta
- [ ] Artículo indexable (aparece en /articulos y sitemap)
- [ ] Enlaces internos correctos
- [ ] Related articles correctos
- [ ] Metadatos completos (title, slug, description, date)
- [ ] No hay duplicados ni errores YAML

---

Este checklist debe ser usado en cada publicación para asegurar calidad y continuidad editorial en Archwise.
